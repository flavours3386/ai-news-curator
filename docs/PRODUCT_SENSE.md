# AI News Curator + LinkedIn Post Generator - Product Sense

## 미션

AI/B2B SaaS/Martech/E-commerce 분야의 핵심 뉴스를 자동으로 수집-분석하고, 전문적인 LinkedIn 포스트를 생성하여 개인 브랜딩과 인사이트 축적을 자동화한다.

## 해결하는 문제

- 매일 20개 이상의 RSS 피드를 수동으로 확인하는 시간 소모
- 관련성 낮은 기사까지 읽어야 하는 정보 과부하
- LinkedIn 포스트 작성에 드는 시간 (기사 선별 -> 관점 정리 -> 작성)
- 뉴스 아카이빙의 체계적 관리 부재

## 사용자

| 사용자 | 역할 | 핵심 니즈 |
|--------|------|-----------|
| 본인 (B2B SaaS 세일즈) | 주 사용자 | 업계 뉴스 자동 큐레이션 + LinkedIn 전문가 포지셔닝 |

## 핵심 가치

1. **자동 큐레이션** — 20개 RSS 피드에서 키워드/AI 기반으로 관련 기사만 자동 필터링하여 노션에 정리
2. **전문가 포스트 생성** — B2B Sales/BizOps 관점의 LinkedIn 포스트를 Claude가 자동 작성하여 초안 제공
3. **비용 효율** — Haiku(필터링) + Sonnet(생성) 모델 분리로 일 1회 실행 기준 월 ~$5 이내 운영

## 제품 원칙

- 사람이 작성한 것처럼 자연스러운 포스트를 지향한다 (AI 냄새 제거)
- 포스트는 항상 "초안" 상태로 저장하고, 최종 게시는 사람이 판단한다
- RSS 소스 추가/삭제는 코드 수정 없이 YAML로 가능해야 한다
- 수집된 뉴스와 생성된 포스트는 모두 노션에 영속 저장한다

## 성공 지표

| 지표 | 현재 | 목표 |
|------|------|------|
| 일일 수집 기사 수 | ~30건 | 20~50건 |
| LinkedIn 포스트 초안/주 | 3개 (일 1회, 회당 최대 3개) | 주 5~10개 |
| 뉴스 관련성 (2차 필터 통과율) | ~40% | 50%+ |
| Railway 크론잡 성공률 | 95%+ | 99%+ |
| 월 AI API 비용 | ~$5 | $10 이내 유지 |

## 경쟁 대안

| 대안 | 장점 | 한계 |
|------|------|------|
| Feedly + 수동 읽기 | 피드 관리 UI 우수 | 필터링/분석 수동, 포스트 생성 없음 |
| Buffer/Hootsuite | SNS 예약 발행 | 뉴스 기반 콘텐츠 자동 생성 불가 |
| ChatGPT 수동 프롬프트 | 유연한 생성 | 자동화 없음, 매번 수동 입력 |
| Zapier + RSS | 자동화 가능 | AI 분석/필터링 부재, 비용 높음 |
